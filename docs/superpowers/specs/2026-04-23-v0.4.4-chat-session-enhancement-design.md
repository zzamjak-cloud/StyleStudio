# v0.4.4 채팅 세션 고도화 설계

## 배경 및 목적

StyleStudio의 채팅 세션을 이미지 중심 대화에서 **이미지 + 기획 문서(PDF/Markdown) 기반 대화**로 확장한다.
주 목적은 **게임 기획 제안서를 바탕으로 첨부된 이미지 또는 문서만으로도 완성도 높은 모바일 캐주얼 게임 인게임 이미지를 생성**하는 것이다.

부가적으로 세션 생성 UX와 저장 폴더 관리, 공통 UI 레이아웃을 단순화한다.

## 범위 (In-Scope)

1. 채팅 세션(BASIC)에 문서 첨부 및 문서 컨텍스트 기반 이미지 생성
2. 새 세션 모달 단순화 (이름 입력 제거, 자동 네이밍)
3. 채팅 세션 아이콘을 STYLE과 구분되게 변경
4. 공통 이미지 분석 화면 상단 고정 레이아웃
5. 저장 폴더 버튼을 "탐색기 열기" 동작으로 변경
6. 세션별 저장 폴더 지정 제거, `~/Downloads/AI_Gen/` 고정
7. 그리드 버튼 4개를 1라인(`grid-cols-4`) 배치
8. 설정 팝업에서 기본 저장 폴더 UI 제거
9. 공통 헤더 높이 축소

## 범위 외 (Out-of-Scope, v0.4.5+)

- **PPT/PPTX 파일 지원**: v0.4.4에서는 제외. 사용자는 PDF 변환 후 첨부 안내.
- PDF 이미지 추출 실패 시 텍스트만으로 폴백 (이미지 품질/정합성 고도화는 추후).
- 세션 자동 네이밍 번호 재사용 로직(삭제된 번호 재사용 등).

## 기능 상세

### 1. 채팅 세션 문서 첨부 (BASIC)

- 기존 `DocumentManager.tsx` 컴포넌트를 `ChatPanel.tsx`에 통합하여 **이미지 + 문서**를 함께 첨부 가능하게 한다.
- 지원 포맷: PDF, Markdown, TXT (기존 `referenceDocument.ts` 파서 재사용).
- **PDF 이미지 추출**:
  - `pdfjs-dist`의 `getOperatorList()` 결과에서 `OPS.paintImageXObject`를 파싱해 내장 이미지 오퍼랜드를 찾는다.
  - `page.objs.get(imageName)`으로 raw `ImageBitmap`/`ImageData` 획득 후 `OffscreenCanvas` → `toBlob('image/png')` → base64 인코딩.
  - 추출된 이미지들은 `ReferenceDocument.extractedImages: string[]` (data URL 배열) 필드에 저장.
- **동작 분기**:
  - `이미지 + 채팅`: 현재 동작 유지.
  - `이미지 + 문서 + 채팅`: AI 요청의 이미지 배열에 `사용자 이미지 + 문서 추출 이미지`를 모두 포함하고, 시스템 프롬프트에 문서 텍스트를 주입.
  - `문서만 + 채팅`: 문서 추출 이미지를 참조 이미지로 사용하고, `"모바일 캐주얼 게임 인게임 이미지 생성"` 프롬프트 템플릿을 자동 주입.
- 문서가 없어도 기존 이미지 기반 대화는 그대로 동작한다 (하위 호환).

### 2. 새 세션 모달 단순화

- `NewSessionModal.tsx`의 세션 이름 입력 필드(63-71행) 제거.
- "시작하기" 버튼 클릭 시 이름 자동 생성.
- **네이밍 규칙 (Option A - 타입별 전체 카운트 기준)**:
  - `{sessionConfig.label}_{해당 타입 세션 총 개수 + 1}` 형식.
  - 예: `기본(채팅)_1`, `기본(채팅)_2`, `캐릭터_1`, `배경_1`.
  - 삭제로 인한 빈 번호는 재사용하지 않는다 (최대값 유지).
- 세션 이름 변경은 기존 세션 리스트의 편집 기능(우클릭/아이콘)으로만 가능.

### 3. 채팅 세션 아이콘 변경

- 현재 BASIC 세션과 STYLE 세션의 아이콘이 시각적으로 혼동됨.
- `NewSessionModal.tsx`에서 BASIC 카드 아이콘을 lucide-react의 `MessageSquare` 또는 `MessagesSquare`로 변경 (STYLE은 `Palette` 유지).
- `sessionConfig.ts`의 `BASIC.icon` 이모지(`💬`)는 세션 리스트/헤더 표시용으로 유지.

### 4. 이미지 분석 화면 상단 고정

- `AnalysisPanel.tsx` 내 "분석 강화", "저장", "생성" 버튼 그룹(194-225행)을 sticky 헤더로 분리.
  - 컨테이너 구조: `<div className="sticky top-0 z-10 bg-white/... border-b">[버튼 그룹]</div>` + `<div className="overflow-y-auto">[분석 프롬프트/결과]</div>`
- 분석 프롬프트 영역만 스크롤되도록 높이 계산.

### 5. 저장 폴더 버튼 동작 변경

- `ImageGeneratorPanel.tsx:1150-1155` 의 `FolderOpen` "저장 폴더" 버튼 클릭 시 `~/Downloads/AI_Gen/`을 OS 파일 탐색기에서 연다.
- 구현: `@tauri-apps/plugin-opener` 또는 기존 `@tauri-apps/plugin-shell`의 `open()` 사용 (플러그인 추가 필요 시 `src-tauri/Cargo.toml` 및 `capabilities/default.json` 갱신).
- 세션별 저장 폴더 지정 UI 제거.

### 6. 저장 경로 고정

- **생성 이미지 저장 경로**: `~/Downloads/AI_Gen/{세션 이름}/`
  - 세션 생성 시 폴더 자동 생성 (`tauri_plugin_fs::create_dir_all`).
  - 세션 이름이 변경되면 기존 폴더 유지 (마이그레이션 복잡도 회피). → 이름 변경 후 새 이미지만 새 이름 폴더로 저장.
- **세션 메타데이터 저장**: `~/Downloads/AI_Gen/Sessions/` (신규 생성).
  - 기존 세션 메타 저장 로직을 이 경로로 마이그레이션.
- **설정 팝업(`SettingsModal.tsx:140-168`)**: 폴더 선택/초기화 버튼 제거, 읽기 전용 안내 텍스트로 교체:
  ```
  생성 이미지는 ~/Downloads/AI_Gen/ 에 저장됩니다.
  ```
  안내 옆에 "폴더 열기" 버튼(↗️)만 제공.

### 7. 그리드 버튼 4개 1라인 배치

- `GeneratorSettings.tsx:212` 의 `grid grid-cols-2 gap-3` → `grid grid-cols-4 gap-2`.
- 버튼 내부 패딩 `p-3` → `p-2`, 폰트 크기 축소(`text-xs` 레벨)로 공간 적응.

### 8. 공통 헤더 높이 축소

- `ImageGeneratorPanel` 및 `AnalysisPanel`의 최상단 헤더 영역 세로 패딩을 `py-4` → `py-2` 수준으로 축소.
- 로고/타이틀 크기도 한 단계 축소 고려.

## 데이터 모델 변경

### `src/types/chat.ts` (기존)

```ts
interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  images?: string[];
  isGeneratedImage?: boolean;
  timestamp: number;
}
```

### `src/types/chat.ts` (변경 후)

```ts
interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  images?: string[];
  documents?: ReferenceDocument[];   // 신규
  isGeneratedImage?: boolean;
  timestamp: number;
}
```

### `src/types/referenceDocument.ts` (확장)

```ts
interface ReferenceDocument {
  // ... 기존 필드
  extractedImages?: string[];   // PDF 내장 이미지 (data URL)
}
```

### 세션 저장 데이터

- 세션 객체에 `saveFolder` 필드가 있다면 deprecated 처리 (읽기만 유지, 신규 저장 시 기록 안 함).
- 기존 세션의 `saveFolder`는 v0.4.4에서 무시하고 `~/Downloads/AI_Gen/{세션명}/` 사용.

## 에러 처리

- PDF 이미지 추출 실패 → 경고 토스트 + 텍스트만으로 계속 진행.
- `~/Downloads/AI_Gen/` 생성 실패 → 에러 토스트 + 폴백 경로(`appDataDir`) 안내.
- 자동 네이밍 시 동일 이름 충돌(매우 드묾) → 접미 타임스탬프 추가.

## 마이그레이션

- 기존 세션의 `saveFolder` 필드는 읽지 않음 (무시). 저장 시에도 기록하지 않음 → 자연 정리.
- 기존 세션 메타데이터는 첫 실행 시 `~/Downloads/AI_Gen/Sessions/`로 이동 시도 (실패 시 기존 경로 유지).
